# 📌 1796. Second Largest Digit in a String

---

# 🧠 Problem Type

* **Category:** String Traversal
* **Pattern:** Track Largest & Second Largest
* **Concept:** Single pass, constant space

---

# 🎯 Core Idea

We need:

* The **second largest distinct digit**
* From digits present in the string

Important:

* Only digits `'0'` to `'9'`
* Ignore letters
* Digits must be **distinct**

---

# 🚀 Optimal Strategy (One Pass)

Use two variables:

```text
max  → largest digit
second → second largest digit
```

Initialize:

```text
max = -1
second = -1
```

For each character:

* If it’s a digit:

  * Convert to integer
  * Update max & second carefully

---

# 🔁 Update Logic

For digit `d`:

### Case 1: New Largest

```
if (d > max)
    second = max
    max = d
```

### Case 2: Between max and second

```
else if (d < max && d > second)
    second = d
```

---

# 💻 Optimal Java Code

```java
class Solution {
    public int secondHighest(String s) {
        int max = -1;
        int second = -1;
        
        for (char ch : s.toCharArray()) {
            if (Character.isDigit(ch)) {
                int digit = ch - '0';
                
                if (digit > max) {
                    second = max;
                    max = digit;
                } 
                else if (digit < max && digit > second) {
                    second = digit;
                }
            }
        }
        
        return second;
    }
}
```

---

# 🧪 Example

Input:

```
"daf12321afd"
```

Digits seen:

```
1, 2, 3
```

Final:

```
max = 3
second = 2
```

Output:

```
2
```

---

# ⏱ Complexity

* **Time:** O(n)
* **Space:** O(1)

Only one traversal. No extra data structures needed.

---

# 🧠 Why Not Use Sorting?

We could:

* Store digits in a Set
* Sort
* Return second last

But that would:

* Use extra space
* Be slower
* Not optimal

Interview prefers constant space solution.

---

# 🚨 Common Mistakes

❌ Not handling distinct condition
❌ Using sorting unnecessarily
❌ Forgetting to ignore letters
❌ Returning max instead of second
