# 📌 26. Remove Duplicates from Sorted Array

---

# 🧠 Problem Type

* **Category:** Array
* **Pattern:** Two Pointers
* **Concept:** In-place array modification

---

# 🎯 Core Goal

Given a **sorted array**, remove duplicates **in-place** such that:

* Each unique number appears **once**
* Return **k = number of unique elements**
* First `k` positions must contain unique elements

Example:

```
[0,0,1,1,1,2,2,3,3,4]
```

Result:

```
[0,1,2,3,4,_,_,_,_,_]
k = 5
```

---

# 🚀 Key Insight

Since the array is **sorted**, duplicates appear **next to each other**.

So we only need to check:

```
nums[i] != nums[i-1]
```

---

# 🧩 Two Pointer Strategy

Use two pointers:

```
i → scans array
k → position of next unique element
```

Steps:

1. First element is always unique.
2. Traverse array from index `1`.
3. If `nums[i] != nums[i-1]`

   * Place element at index `k`
   * Increment `k`.

---

# 💻 Optimal Java Code

```java
class Solution {
    public int removeDuplicates(int[] nums) {
        int k = 1; // first element always unique
        
        for (int i = 1; i < nums.length; i++) {
            if (nums[i] != nums[i - 1]) {
                nums[k] = nums[i];
                k++;
            }
        }
        
        return k;
    }
}
```

---

# 🧪 Example Walkthrough

Input:

```
[0,0,1,1,1,2,2,3,3,4]
```

Process:

```
k = 1

i = 1 → duplicate → skip
i = 2 → new element → nums[1] = 1
i = 3 → duplicate
i = 4 → duplicate
i = 5 → nums[2] = 2
i = 7 → nums[3] = 3
i = 9 → nums[4] = 4
```

Result:

```
[0,1,2,3,4,...]
k = 5
```

---

# ⏱ Complexity

* **Time:** O(n)
* **Space:** O(1)

No extra memory used.

---

# 🏆 Why Two Pointers Work

Because array is **already sorted**.

Duplicates are adjacent → easy to detect.

---

# 🚨 Common Mistakes

❌ Using extra array
❌ Using HashSet unnecessarily
❌ Forgetting array is sorted
