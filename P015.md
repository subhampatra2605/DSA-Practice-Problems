# 🏁 3668. Restore Finishing Order

---

# 🧠 Problem Type

* **Category:** Arrays
* **Pattern:** Filtering / Membership check
* **Concept:** Preserve order while selecting subset

---

# 🎯 Core Understanding

* `order` → Final race finishing order (permutation of 1 to n)
* `friends` → IDs of your friends (sorted, but sorting not needed)

We must:

> Return friends in the same order they appear in `order`.

⚠️ Important:
We do NOT sort.
We do NOT rearrange.
We just preserve `order`.

---

# 🔁 Algorithm (Optimal Approach)

1. Put all `friends` into a `HashSet`.
2. Traverse `order`.
3. If element exists in set → add to result.
4. Return result array.

---

# 💻 Clean Implementation Idea

```
Create HashSet of friends
For each id in order:
    if id exists in set:
        add to result
Return result
```

---

# ⏱ Complexity

* **Time:** O(n)
* **Space:** O(f) (max 8)

Since:

* `n ≤ 100`
* `friends.length ≤ 8`

Even O(n × f) brute force works.

---

# 🧩 Why HashSet?

* Membership check becomes O(1)
* Cleaner logic
* Interview-friendly

---

# 📌 Key Insight

This is NOT:

* Sorting
* Two pointers
* Binary search

It’s simply:

> “Filter while preserving original order”

---

# 🏆 Interview Explanation (Short Version)

> I store friends in a HashSet for constant-time lookup. Then I iterate through the finishing order and collect IDs that are in the set. This preserves the finishing order automatically.

---


# 🧠 Pattern Tag

Remember this as:

👉 **“Filter by membership while preserving sequence order”**
