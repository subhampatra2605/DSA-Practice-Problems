# 📌 396. Rotate Function

---

# 🧠 Problem Type

* **Category:** Array + Math
* **Pattern:** Optimization / Recurrence Relation
* **Concept:** Reusing previous computation (DP-like)

---

# 🎯 Core Goal

Given an array `nums`, define:

[
F(k) = \sum_{i=0}^{n-1} i \cdot arr_k[i]
]

Where:

* `arr_k` = array rotated **k times clockwise**

👉 Return the **maximum value** among all `F(k)`.

---

# 🚀 Key Insight

Brute force:

* Rotate array each time → O(n)
* Compute F(k) → O(n)

👉 Total:

```text
O(n²) ❌
```

---

# 🔥 Critical Observation

Instead of recomputing every time, derive:

[
F(k) = F(k-1) + \text{sum(nums)} - n \cdot nums[n-k]
]

---

# 💡 Intuition Behind Formula

When rotating:

* Every element shifts right → contributes **+1 weight**
* The last element moves to front → loses **(n-1) weight**

👉 Net effect:

```text
Add sum(nums)
Subtract n × last element moved
```

---

# 🧩 Optimized Strategy

1. Compute:

```text
sum = sum(nums)
F(0) = Σ i * nums[i]
```

2. Use recurrence:

```text
F(k) = F(k-1) + sum - n * nums[n-k]
```

3. Track maximum

---

# 💻 Optimal Java Code

```java
class Solution {
    public int maxRotateFunction(int[] nums) {

        int n = nums.length;

        long sum = 0;
        long F = 0;

        // Step 1: compute sum and F(0)
        for (int i = 0; i < n; i++) {
            sum += nums[i];
            F += (long) i * nums[i];
        }

        long max = F;

        // Step 2: compute F(k)
        for (int k = 1; k < n; k++) {
            F = F + sum - (long) n * nums[n - k];
            max = Math.max(max, F);
        }

        return (int) max;
    }
}
```

---

# 🧪 Example Walkthrough

Input:

```
nums = [4,3,2,6]
```

Step 1:

```
sum = 15
F(0) = 25
```

Step 2:

```
F(1) = 25 + 15 - 4*6 = 16
F(2) = 16 + 15 - 4*2 = 23
F(3) = 23 + 15 - 4*3 = 26
```
