# ЁЯУМ 788. Rotated Digits

---

# ЁЯза Problem Type

* **Category:** Math / Digit Processing
* **Pattern:** Digit Validation + Transformation
* **Concept:** Classification of digits (valid vs invalid vs changing)

---

# ЁЯОп Core Goal

Count numbers in range `[1, n]` such that:

* Every digit is **valid after rotation**
* The resulting number is **different from original**

---

# ЁЯФД Digit Rotation Rules

| Digit | After Rotation | Type      |
| ----- | -------------- | --------- |
| 0     | 0              | same      |
| 1     | 1              | same      |
| 8     | 8              | same      |
| 2     | 5              | different |
| 5     | 2              | different |
| 6     | 9              | different |
| 9     | 6              | different |
| 3,4,7 | тЭМ invalid      | invalid   |

---

# ЁЯЪА Key Insight

A number is **GOOD** if:

```text
1. All digits are valid
2. At least one digit changes after rotation
```

---

# тЭМ Invalid Case

If any digit тИИ `{3,4,7}`
ЁЯСЙ number is invalid

---

# тЪая╕П Not Good Case

If all digits тИИ `{0,1,8}`
ЁЯСЙ number remains same тЖТ NOT good

---

# ЁЯзй Strategy

For each number from `1 тЖТ n`:

1. Check digits one by one
2. Track:

   * `valid = true/false`
   * `changed = true/false`
3. Count if:

```text
valid == true AND changed == true
```

---

# ЁЯТ╗ Optimal Java Code

```java
class Solution {
    public int rotatedDigits(int n) {

        int count = 0;

        for (int i = 1; i <= n; i++) {
            if (isGood(i)) count++;
        }

        return count;
    }

    private boolean isGood(int num) {

        boolean changed = false;

        while (num > 0) {

            int d = num % 10;

            if (d == 3 || d == 4 || d == 7) {
                return false; // invalid
            }

            if (d == 2 || d == 5 || d == 6 || d == 9) {
                changed = true;
            }

            num /= 10;
        }

        return changed;
    }
}
```
